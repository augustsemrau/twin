# Twin — Personal Context Engine

## What this project is

Twin is a Tauri 2.x desktop app that maintains a live, structured model of a knowledge worker's work state (the "work graph") and dispatches tailored context packs to Claude Chat, Claude Code, and Claude Cowork. The core hypothesis: structured context packs make AI sessions meaningfully better with near-zero user overhead.

**Spec:** `twin-spec-v1.1.md` is the authoritative reference. Read it before making architectural decisions.

## Architecture

- **Filesystem-canonical, graph-derived.** YAML and markdown files in `~/twin/` are the source of truth. The work graph is derived in-memory and never persisted separately.
- **Structured data in YAML** (`tasks.yaml`, `deliveries.yaml`, `decisions.yaml`, `people.yaml`), **prose in markdown** (`notes/*.md`, `context.md`).
- **Every entity has a ULID.** References between entities use IDs, never display strings.
- **All state changes go through typed delta operations** validated by the Validator and applied by the State Updater. No agent writes to canonical files directly.

## Tech stack

- **App shell:** Tauri 2.x (Rust backend, WebView frontend)
- **Frontend:** React 18 + TypeScript + Vite + Tailwind CSS
- **Graph viz:** @antv/g6 v5 (WebGL, force-directed, combo nodes)
- **YAML:** eemeli/yaml (round-trip with comment preservation — comments above keys only, never inline)
- **Frontmatter:** gray-matter
- **Markdown:** markdown-it (CommonMark compliant, safe defaults)
- **AI:** @anthropic-ai/sdk (Haiku 4.5 for Resolver/Planner/Reconciler, Sonnet 4.5 for Prioritiser/Composer)
- **IDs:** ulid
- **Fuzzy match:** uFuzzy (short-phrase matching, transparent scoring)
- **Clipboard:** tauri-plugin-clipboard (CrossCopy)
- **macOS permissions:** tauri-plugin-macos-permissions

## Key conventions

- **YAML comments:** above the relevant key, never inline/trailing (eemeli/yaml round-trip stability)
- **Delta operations:** always reference entities by ULID, never by title string
- **Decision lifecycle:** `active | superseded`. Composer includes only `active` decisions in context packs.
- **CLAUDE.md generation:** on dispatch (if stale) or explicit request — never on every file change
- **Confidence:** categorical `high | medium | low` — never numeric scores
- **File writes:** always check mtime before writing (optimistic concurrency control)
- **API calls:** retry 3x with exponential backoff (1s, 4s, 16s), 30s timeout, graceful degradation on failure

## Folder structure

```
~/twin/
├── CLAUDE.md              # Global context (user role, style, toolchain)
├── people.yaml            # People model
├── inbox/                 # Unprocessed captures
├── sessions/              # Context packs + manifests
│   └── writeback-schema.yaml
├── archive/               # Archived projects and old sessions
└── projects/
    └── [slug]/
        ├── CLAUDE.md      # Generated by Twin — do not edit
        ├── context.md     # Background, client, goal, constraints
        ├── tasks.yaml     # Task list with ULIDs
        ├── deliveries.yaml
        ├── decisions.yaml  # Decision log with active/superseded lifecycle
        └── notes/
```

## Internal agents

| Agent | Model | Purpose |
|---|---|---|
| Resolver | Haiku | Convert captures/imports into typed observations |
| State Updater | None (mechanical) | Apply validated deltas to files |
| Planner | Haiku | Recommend dispatch target and sources for an objective |
| Prioritiser | Sonnet | Generate daily focus brief and proactive proposals |
| Composer | Sonnet | Assemble context packs with entity ID mapping |
| Reconciler | Haiku | Turn session manifests into proposed deltas |
| Validator | None (rule-based) | Check delta operations before State Updater applies them |

## The `fs.ts` layer

All filesystem operations go through `fs.ts`. Components never call Tauri's fs plugin directly. This is the single seam for storage changes.

## Important constraints

- External agents never mutate canonical state directly — they write manifests
- No silent writes to canonical state — all changes are proposed then confirmed
- Capture must never be blocked by AI processing — file write happens first, Resolver runs async
- Quick dispatch (`Cmd+D`) must complete in under 30 seconds
- The visual graph is a core differentiator, not a nice-to-have
- Chat writeback adoption is the biggest risk — track which path is used per session
- Archived projects are excluded from the work graph entirely
